# 媚吻锋 Agent 工具升级战略规划

为提升媚吻锋（AI导师）作为智能体（Agent）的任务处理能力与稳定性，特制定“四步走”升级计划。本计划需严格遵循“开发一步、测试一步、验收一步”的原则推进。

## 第一步：高效搜索文件与多任务并发 [已达成]
**目标**：彻底解决模型依赖低效 Bash 盲搜的问题，并解除“一次只能执行一个工具”的封印。
- **并发工具调用**：修改 Prompt 与模型后端路由逻辑，允许模型在单次输出中抛出多个 `<call_tool>`，并能在前端同时渲染动画与返回多个结果。
- **全局内容检索 (Grep Search)**：新增专用的语义/文本检索工具。通过输入指定目录和关键词，系统能极速匹配所有相关文件及行号，极大提升查阅设定、代码和文档的效率。

## 第二步：API 访问重试与防阻断机制 [已达成]
**目标**：提升弱网或高并发环境下的对话稳定性。
- **拦截与重试**：模仿 Claude Code 等主流终端 AI 体验，在模型底层 API 限制访问频率或断线时（如 `ConnectionRefused` 等报错），实现拦截与退避重试（Exponential Backoff）。
- **用户友好提示**：在前端对话气泡下方使用小字标注重试状态（如 `Retrying in 3s · attempt 4/10`），消除用户的等待焦虑。

## 第三步：赋予“沙盒演进”级别的写功能 [已达成]
**目标**：并非为了让 AI 代替你写项目代码，而是为了实现**“可持续演进的测试沙盒”**。
- **演进式测验 (replace_file_content)**：当 AI 生成了一个 HTML 小程序或 Python 测试脚本（例如 `docs/sandbox/test_01.html`）供你练习时，如果你答对了第一关，AI 可以直接修改这个文件，增加新的难度或引入新的 Bug 让你继续排查，而不是每次都生成一堆垃圾文件。
- **沙盒隔离机制**：这把“手术刀”的权限将被严格约束在特定的教学沙盒目录中，有效防止幻觉导致的项目核心代码（如 `App.jsx`）被意外破坏。

## 第四步：轻量级前端交互测验 (替代原有重度 Agent 路线) [已达成]
**目标**：放弃重度的后台挂机（Background Tasks）与子代理（Sub-agents），拥抱轻量级的全栈教学闭环。
- **UI 原生测验组件**：对于不需要编写代码的文科/基础理论测验，模型直接在对话气泡中吐出 JSON，前端渲染为点选式的互动组件，做到秒级反馈，杜绝过度依赖系统文件操作带来的沉重感。

## 第五步：联网检索与外部知识获取 (当前阶段)
**目标**：为大模型插上联网的翅膀，突破知识库边界。
- **纯粹与轻量**：使用 ddgs (原 duckduckgo-search) 提供免费且开箱即用的检索能力。
- **沉浸感隔离**：模型搜索网页的工具调用及冗长的 HTML/文本抓取结果，必须在 AgentExecutor 层被完全拦截和静默处理，绝不能流出到前端主聊天气泡中，誓死捍卫 Roleplay 沉浸感。
- **上下文保护**：限制搜索返回的链接数与摘要长度，防止瞬间爆显存或触发 unexpected EOF 中断。
