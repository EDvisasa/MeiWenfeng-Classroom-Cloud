# 媚吻锋随身课堂 —— 开发计划总纲

> **核心规范与纲领说明**：本文件是整个项目的宏观规划蓝本，只记录核心架构、设计决策与宏观阶段规划。具体的待办任务（Checklist）和子系统设计，请参阅本目录下的其他子项文件。
> **文档与架构遵守铁律**：全站所有架构设计文档、目录结构与命名准则必须服从最高总纲 **`docs/00_文档规范与命名总纲.md`** 的约束，恪守 **角色扮演系统（台词人设模拟）** 与 **严肃教学及底座系统（现代工程技术标准）** 的双领域物理隔离。

## 一、核心架构与设计决策

| 维度 | 最终决定 |
|---|---|
| 后端语言 | **Python (FastAPI)** |
| 前端框架 | **React + Vanilla CSS** |

## 三、底层数据库表设计 (SQLite 全量真实映射)

严格恪守技术底座现代命名与数据建模规范，当前系统真实存在的 11 张核心数据表定义如下：

- **会话与基础消息表**：
  - `chat_sessions`: 会话历史记录会话元数据（ID、标题、当前激活模型等）
  - `chat_messages`: 每一轮的具体对话与角色气泡文本
- **教学与课程状态表**：
  - `course_progress`: 严肃教学系统的进阶课程大纲与用户掌握状态（ZPD 追踪）
  - `glossary`: 核心课程术语字典表（基于 Matt Pocock 理念）
  - `learning_decision_records`: 学习决策记录（LDRs，记录证据 `evidence` 与推论）
- **长期学习目标管理表**：
  - `user_mission`: 用户确立的全局学习目标与核心任务指标
  - `mission_draft`: 目标拟定期的临时数据缓冲与锁定状态
- **系统引擎与检索配置表**：
  - `model_config`: 多模型接入配置及 API Key 密钥池
  - `rag_config`: 动态 RAG 检索后端连接参数（ChromaDB / RAGFlow）
- **角色扮演记忆表**：
  - `memory_logs`: 长期记忆日志（供四段式记忆衰减脚本处理）
  - `affection`: 虚拟导师交互好感度与亲密度参数（在 UI 面板中隐蔽折算）

## 四、斜杠命令流转体系

| 命令 | 后端行为 | 目标 |
|---|---|---|
| `/lesson` | 切入"授课模式" | 根据 ZPD 推送包含测验组件的交互微课 |
| `/submit` | 切入"考核模式" | 校验答案，触发 Feedback Loop 或 `execute_bash` |
| `/set_mission`| 设定大目标 | 将宏大目标存入数据库并常驻状态栏 |
| `/summarize` | 压缩记忆 | 触发四段式衰减（memory_decay.py） |
| `/prepare` | 同步知识库 | 将选定的本地文件上传至 ChromaDB/RAGFlow |
| `/plan` | 制定学习计划 | 基于大纲进度，输出下一步的栽培任务 |

## 五、宏观开发阶段定义

### 第一阶段 · MVP（已完成）
建立基础三栏布局、FastAPI 后端、连通模型并实现流式输出与人设挂载。

### 第二阶段 · 知识接入与底层基建（已完成）
实现本地 ChromaDB、本地文件树挂载、自动同步解析，以及对话内容的向量化记忆。

### 第三阶段 · 课堂引擎与 Agent 底层融合（当前开发阶段）
**核心目标**：将系统从“聊天机器人”升级为具备“闭环反馈与长程记忆”的教学智能体。
- 重点 1：补齐 Agent 底层工具（精准读写与抗压重试）。
- 重点 2：引入交互式前端组件，废弃纯文本灌输。
- 重点 3：建立基于 `/lesson` 和 `/submit` 的真实闭环反馈。

### 第四阶段 · 高级扩展（未来阶段）
独立换肤、后台异步任务挂载、完整的原生协议路由。
