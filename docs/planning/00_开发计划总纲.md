# 媚吻锋随身课堂 —— 开发计划总纲

> **核心文件说明**：本文件是整个项目的唯一“纲领文件”，只记录核心架构、设计决策与宏观阶段规划。具体的待办任务（Checklist）和子系统设计，请参阅本目录下的其他子项文件。

## 一、核心架构与设计决策

| 维度 | 最终决定 |
|---|---|
| 后端语言 | **Python (FastAPI)** |
| 前端框架 | **React + Vanilla CSS** |
| 数据存储 | **SQLite**（记忆日志、课程进度、好感度、模型配置） |
| 知识库 | **RAGFlow 本地实例** / **ChromaDB 本地内嵌库** |
| UI 布局 | **三栏 IDE 式**：左(文件树) / 中(对话授课区) / 右(状态面板) |
| 交互特色 | **斜体灰色动作描写**，好感度隐性化处理 |

## 二、多模型支持矩阵

| 模型 | 接口协议 | 默认地址 |
|---|---|---|
| DeepSeek (在线) | OpenAI 兼容 | `api.deepseek.com` |
| Gemini-Antigravity (在线) | Gemini 协议 | `generativelanguage.googleapis.com` |
| Gemma-9b (本地) | OpenAI 兼容 | `http://localhost:11434` (Ollama) |

## 三、底层数据库表设计 (SQLite)

- `memory_logs`: 记忆日志（带时间戳，供衰减脚本处理）
- `course_progress`: 课程大纲与用户理解进度 (ZPD 追踪)
- `affection`: 好感度系统（隐蔽，不在 UI 直接展示）
- `model_config`: 多模型与 API Key 配置池

## 四、斜杠命令流转体系

| 命令 | 后端行为 | 目标 |
|---|---|---|
| `/lesson` | 切入"授课模式" | 根据 ZPD 推送包含测验组件的交互微课 |
| `/submit` | 切入"考核模式" | 校验答案，触发 Feedback Loop 或 `execute_bash` |
| `/set_mission`| 设定大目标 | 将宏大目标存入数据库并常驻状态栏 |
| `/summarize` | 压缩记忆 | 触发四段式衰减（memory_decay.py） |
| `/prepare` | 同步知识库 | 将选定的本地文件上传至 ChromaDB/RAGFlow |
| `/plan` | 制定学习计划 | 基于大纲进度，输出下一步的栽培任务 |

## 五、宏观开发阶段定义

### 第一阶段 · MVP（已完成）
建立基础三栏布局、FastAPI 后端、连通模型并实现流式输出与人设挂载。

### 第二阶段 · 知识接入与底层基建（已完成）
实现本地 ChromaDB、本地文件树挂载、自动同步解析，以及对话内容的向量化记忆。

### 第三阶段 · 课堂引擎与 Agent 底层融合（当前开发阶段）
**核心目标**：将系统从“聊天机器人”升级为具备“闭环反馈与长程记忆”的教学智能体。
- 重点 1：补齐 Agent 底层工具（精准读写与抗压重试）。
- 重点 2：引入交互式前端组件，废弃纯文本灌输。
- 重点 3：建立基于 `/lesson` 和 `/submit` 的真实闭环反馈。

### 第四阶段 · 高级扩展（未来阶段）
独立换肤、后台异步任务挂载、完整的原生协议路由。
