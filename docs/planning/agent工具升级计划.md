# 媚吻锋 Agent 工具升级战略规划

为提升媚吻锋（AI导师）作为智能体（Agent）的任务处理能力与稳定性，特制定“四步走”升级计划。本计划需严格遵循“开发一步、测试一步、验收一步”的原则推进。

## 第一步：高效搜索文件与多任务并发 (当前阶段)
**目标**：彻底解决模型依赖低效 Bash 盲搜的问题，并解除“一次只能执行一个工具”的封印。
- **并发工具调用**：修改 Prompt 与模型后端路由逻辑，允许模型在单次输出中抛出多个 `<call_tool>`，并能在前端同时渲染动画与返回多个结果。
- **全局内容检索 (Grep Search)**：新增专用的语义/文本检索工具。通过输入指定目录和关键词，系统能极速匹配所有相关文件及行号，极大提升查阅设定、代码和文档的效率。

## 第二步：API 访问重试与防阻断机制
**目标**：提升弱网或高并发环境下的对话稳定性。
- **拦截与重试**：模仿 Claude Code 等主流终端 AI 体验，在模型底层 API 限制访问频率或断线时（如 `ConnectionRefused` 等报错），实现拦截与退避重试（Exponential Backoff）。
- **用户友好提示**：在前端对话气泡下方使用小字标注重试状态（如 `Retrying in 3s · attempt 4/10`），消除用户的等待焦虑。

## 第三步：赋予“外科手术刀”级别的写功能
**目标**：改变此前只能凭借 `bash echo` 覆写全文件的危险局面，实现对文件的精准打补丁。
- **精准替换工具 (replace_file_content)**：要求模型严格提供目标文件的绝对路径、想要替换的【旧文本内容】和替换的【新文本内容】。
- **安全检查机制**：强制进行文本唯一性验证，如果不唯一或找不到，则驳回修改请求，有效防止幻觉导致的代码或小说设定被破坏。

## 第四（五）步：加强大型复杂任务的处理能力
**目标**：赋予她接手大型跨文件、跨系统任务（如整包重构、互联网资料搜集整理）的能力。
- **后台异步任务 (Background Tasks)**：如果某些搜索或代码测试需要运行 30 秒甚至更久，模型不应堵塞对话。增加 `manage_task` 与定时机制，让模型把耗时任务扔进后台，并在完成后主动通过“计划调度（schedule）”向用户汇报。
- **子代理委托 (Sub-agents)**：在遇到自身当前环境或上下文无法解决的繁杂事务（如网页检索总结）时，能拉起轻量级的下属子代理，子代理完成后直接向她做工作汇报。*(此功能高度依赖底层 API 对多模型的并发支持，具体细节将在第三步完成后详细论证)*。
