## 目标概述

打造一款完全独立于 SillyTavern 的本地 Web 应用。以 Python 后端 + React 前端构建，通过 SQLite 数据库和 RAGFlow 本地知识库，将"好感度"、"课程进度"、"记忆管理"全部从 LLM Prompt 中剥离，实现极限 Token 压缩。

## SQLite 数据库表设计

```sql
-- 记忆日志（带时间戳，供衰减脚本处理）
CREATE TABLE memory_logs (
    id INTEGER PRIMARY KEY,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    content TEXT,           -- 原始内容
    summary TEXT,           -- 压缩摘要（衰减后填充）
    level INTEGER           -- 0=精确 1=关键 2=模糊 3=里程碑 4=索引
);

-- 课程大纲与进度
CREATE TABLE course_progress (
    id INTEGER PRIMARY KEY,
    phase TEXT,             -- 阶段名称
    topic TEXT,             -- 课题名
    status TEXT,            -- pending / active / done
    score INTEGER           -- 用户理解程度 0~100
);

-- 好感度（完全后台，不在UI展示）
CREATE TABLE affection (
    id INTEGER PRIMARY KEY DEFAULT 1,
    value INTEGER DEFAULT 50,   -- 0~100
    last_updated DATETIME
);

-- 模型配置
CREATE TABLE model_config (
    id INTEGER PRIMARY KEY,
    name TEXT,
    protocol TEXT,          -- openai / claude / gemini
    base_url TEXT,
    api_key TEXT,
    is_active INTEGER DEFAULT 0
);
```

## 记忆衰减规则

| 时间范围 | 精度等级 | 处理方式 |
|---|---|---|
| < 3 天 | 精确 (level 0) | 完整对话记录，全文传入上下文 |
| 3~7 天 | 关键 (level 1) | 仅保留关键事件节点（AI 自动压缩） |
| 7~30 天 | 模糊 (level 2) | 生成 1~2 句话的简短印象 |
| 30~360 天 | 里程碑 (level 3) | 仅保留高度压缩的里程碑记录 |
| > 360 天 | 索引 (level 4) | 仅保留工程路径和时间线标题，供用户自查 |

衰减脚本在每次 `/总结` 指令或每日首次启动时触发。

## 斜杠命令说明

| 命令 | 触发逻辑 | 后端行为 |
|---|---|---|
| `/上课` | 开始新课时 | 加载当前课时大纲，切换 System Prompt 为授课模式 |
| `/交作业` | 检验理解程度 | AI 切换为"考核模式"，出题并等待用户回答 |
| `/奖励` | 课后奖励 | 读取好感度，AI 进入娇妈娇互动模式，解锁奖励段落 |
| `/总结` | 压缩记忆 | 触发 memory_decay.py，压缩当前对话存入 SQLite |
| `/切换模型` | 切换 LLM | 前端弹出模型选择，后端切换 model_router 目标 |
| `/备课` | 上传知识库 | 后端调用 RAGFlow API，将左侧选定文件夹上传解析 |
| `/计划` | 调整大纲 | AI 读取历史进度，对课程大纲尾部进行重新规划 |
