动态人设注入与系统提示词管线决策

收纳合并原 agent_prompt_analysis、persona-prompt-logic 与 persona-real-prompt-structure 审查报告。基于 Matt Pocock “高信噪比”理念，解决多性格虚拟导师提示词冗长、XML 标签回显污染 UI 及提示词越狱问题。

🏷️ LLM-Prompt-Injection (人设与系统提示词注入拦截) ✨ Roleplay-Immersion (酒馆式角色扮演沉浸) 🏷️ XML-Tag-Enforcement (结构化标签解析)
1

背景与技术挑战 (Context & Challenges)

静态冗杂提示词导致的指令遗忘与前端标签污染

挑战一：巨型硬编码 Prompt 导致注意力稀疏

旧版系统将所有角色的性格设定、教学规则、格式要求堆叠在一个几千字的硬编码字符串中，LLM 在多轮对话后经常遗忘教学约束，发生“人设坍塌”。

挑战二：教学功能组件与对白混淆

需要 AI 输出讲解卡片或测试题时，若不强制结构化约束，AI 往往在自然对白中夹杂不稳定格式，前端难以正则提取，导致 UI 渲染破损。

2

决策方案与深模块解耦 (Decision & Solution)

动态组装管线与 XML 强制收口

2.1 动态提示词管线与人设加载 (load_local_persona & get_system_prompt)

在 prompts.py 中建立深模块函数 load_local_persona("simplified") 读取本地缓存 mei_wenfeng_persona_simplified.txt，并通过 get_system_prompt 按层级动态组装：Base System Rule -> Character Persona -> Current Mission Context，大幅降低 Token 噪声。

2.2 严格 XML 标签通信协议

强制约束 LLM 使用专用标签输出功能组件：<explainer>（生成讲义卡片）、<quiz>（生成互动测验）、<glossary>（高亮专业词汇）。后端流式解析器拦截这些标签并静默转换为 RAG 引用材料或 UI 卡片，绝不泄露给前端对白流。

  • ✓ 彻底杜绝标签污染：前端用户只能看到生动的导师教学对白，互动组件无缝以高级 UI 卡片弹出。
