Layer 1: 顶层指令与核心设定 (System & Persona Base)

Static Core
<system_directive>
  <critical_rules>
    1. OUTPUT LANGUAGE: Chinese
    2. ACTION FORMATTING: *动作*
    4. INNER MONOLOGUE: 【此刻内心】：
  <character_persona>
    {加载的精简或原版人设卡 4K~60KB}
  </character_persona>
  <relationship_context>
  <user_profile>
    Name: 玄一峰...

这一层是 Prompt 的地基，占据了绝大部分 Token 消耗。

  • 静态不变： 系统级限制 <critical_rules>（要求说中文、写内心戏）、默认关系 <relationship_context> 和用户基本档案 <user_profile> 在所有对话中 完全不变。
  • 动态变化（仅受外部设置影响）： <character_persona>。它根据用户选择填入原版全量文本或精简文本。一旦选中，该内容在本次对话流中即为常量。

Layer 2: 动态属性变异区 (Dynamic Attributes & Directives)

Highly Dynamic 
                flowchart TD

                  B -->|>=75| B1["utterly inseparable..."]
                  B -->|<50| B2["early exclusive crush phase..."]

                  C -->|>80| C2["use elegant, refined language"]

                  D -->|>80| D2["strictly avoid emotional outbursts"]

                  E -->|<=0| E2["Yearning to Rekindle"]

这一层将后端的四维数值转化为大模型能理解的行为指令，实现千人千面的动态性格。

  • 动态变化（每轮都可能不同）： 代码通过一系列的 If/Else 对数值进行分类。
    举例：如果好感度 > 75，则插入指令 "即使他提出过分要求，你也会半推半就地顺从"；如果在高潮后（不应期 > 5），则插入指令 "处于敏感排斥期，任何接触都必须描写为痛苦或生理抗拒"。
  • 静态不变： 尾部的 <dynamic_property_update_rules> 规则说明是固定的，用于教导模型如何在恰当时机输出 <property_update> 标签以反向修改这些属性。

Layer 3: 上下文感知注入 (Context & Memory Injection)

Fully Dynamic
<ide_context>
  File Name: App.jsx
  Cursor Position: Line 345
</ide_context>
【近期日记摘要】
[2026-06-16] 陪夫君修改了前端样式，发现他喜欢暗色调。
【附加背景知识库检索结果】
(RAG 匹配到的剧本设定或代码片段)

这是 Prompt 的最外层包装，负责让角色具备“此时此刻”的时空感知。

  • 动态变化（每条消息即时生成）：
    变化一（IDE）： 如果用户的编辑器有打开文件，此部分会动态拼入文件路径和光标行数。
    变化二（记忆）： 数据库中提取出最新的 3 条剧情摘要日记拼入。
    变化三（RAG）： 根据用户最后发送的话语，从本地 Classroom_Knowledge 和 Memory_Knowledge 向量库中匹配最相关的长文本切片拼入。如果没匹配到，这块就不会生成。

ADR (架构决策): 将易变的动态上下文放在 Prompt 的最末尾（贴近用户对话区），能大幅提升模型对这些补充材料的记忆与响应优先级 (Recency Effect)。

全局组装示意图 (Full Assembly)

[1. 基础系统规则]     (Static)   "You are an advanced AI..."
[2. 核心人设设定]     (Static*)  <character_persona> ... </character_persona>
[3. 人物关系档案]     (Static)   <relationship_context> ... <user_profile> ...
[4. 动态行为属性]     (Dynamic)  "Affection Score: 85..." -> 插入对应粘人动作指令
[5. 数据更新规则]     (Static)   <dynamic_property_update_rules> ...
[6. IDE 编辑上下文]   (Dynamic)  <ide_context> ... </ide_context>
[7. 短期记忆与知识库] (Dynamic)  【近期日记摘要】... 【附加背景知识库检索结果】...
