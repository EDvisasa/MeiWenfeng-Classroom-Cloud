将浅层代码收敛为六大深模块解耦工程

确立自上而下的六层工程拓扑结构 (Hierarchical Layering)，统筹 ADR-001 ~ ADR-007 架构决策记录。通过物理切片屏障与深模块收口，彻底消除跨架构接缝的状态泄漏 (state leakage) 与浅层过度包装 (shallow wrapping)。

实线框 = 独立模块 虚线 = 架构接缝 红色箭头 = 状态泄漏 深色厚框 = 深模块

核心架构哲学与限界上下文物理隔离 (Domain Bounded Contexts)

本工程由两个本质不同、高度解耦的核心子系统交织而成。在底层源码、API 规范、系统日志与架构文档中，严禁用词与职责混淆，必须严格恪守领域驱动设计 (DDD) 的物理界限：

🎭 角色扮演系统 (Roleplay Persona System)

专属领域：虚拟人物模拟器、情感互动与对话语境映射。
合法合规用词：允许且推荐使用人设模拟器专属参数，如 CSM 状态机、好感度 (affection)、阶位 (social_status)、性格提示词 等。

💻 严肃教学与底座系统 (Serious Teaching & Core Architecture)

专属领域：教学大纲调度、讲义文件落地、流式通信管线与异步 I/O 事务。
合法合规用词：必须严格使用现代、准确、高效、凝练的技术软件工程术语（如 讲义 Markdown 文件持久化、SSE 流式生成、架构决策记录）。严禁在此领域混入国风修仙或小说叙事词汇。

演进背景与浅层泄漏痛点 (Context & Challenges)

核心痛点：早期的 API 路由与前端组件多为“浅模块 (Shallow Modules)”——接口表面的复杂性几乎等于内部实现的复杂性，导致业务逻辑与裸写 SQL 跨越架构接缝四处泄漏，引发接口阻塞与通信死锁。

泄漏接缝点：frontend/src/App.jsx · backend/routers/chat.py · backend/services/agent_tools.py

架构剖面对比：浅层泄漏 vs 深模块收口
重构前：浅层过渡包装（状态四处泄漏）
前端层：流式对白渲染与 XML 标签解析混杂
路由层：裸写 SQL 与 LLM 同步调用阻塞
工具层：同步阻塞式网页爬取与无界分页
重构后：深模块封装（极简接口，极高杠杆效益）
L1 前端沉浸屏障 blockParser 切片屏障
L4 深模块生态与状态机 _state_lock 事务锁 / 800行硬截断

决策方案与深度重构 (Decision & Solution)

架构对策：设立严格的自上而下六层边界，让深模块内部吸收极度复杂的业务逻辑；前端切片屏障物理隔离对白与 UI 卡片，后处理响应管线异步解耦重度副作用。

后果评估与职责收敛 (Consequences & Evaluation)

架构升格关键收益
✓ 局部性：缺陷锁定单一模块
✓ 杠杆力：一处接口多处复用
✓ 收接口：深层吸收浅层包装
✓ 强接缝：严格隔离状态泄漏
✓ 事务锁：彻底消除并发死锁
✓ 异步化：释放主服务响应流
工程源码职责矩阵对照表
层级划分 核心源码文件 对应 ADR 规范 设计模式与工程机制
L1 前端层 blockParser.js ADR-007 流式物理切片屏障，将纯文字对白与互动 UI 卡片隔离
L2 路由层 slash_handler.py ADR-004 快捷指令前置分流中枢，互动卡片与平滑上下文引导语注入
L3 决策层 prompts.py ADR-002 动态提示词管线，实时拉取角色好感度依赖注入与 XML 通信协议
L4 深模块 agent_tools.py ADR-001 免Key泛搜 3 条封顶、DOM 降维清洗、800 行物理硬截断
L4 深模块 character_state.py ADR-003 CSM 角色状态机单例封装，内部由 _state_lock 线程锁收口事务
L5 异步层 response_pipeline.py ADR-003 发布订阅模式，实现主请求毫秒级返回与副作用解耦异步写入讲义文件
L5 异步层 memory_decay.py ADR-005 时间梯度衰减浓缩引擎 (-2d/-7d/-30d/-365d) 与四字段归纳
L6 存储层 materials_manager.py ADR-005 RAG 知识库检索白名单拦截 (严禁脏数据，仅扫描白名单目录)
