问题痛点： App.jsx 目前是一个浅模块（Shallow Module）—— 具体的实现逻辑直接暴露在了 UI 层。它甚至明确知道 if (textToSend === '/summarize') 这种具体的指令字符串，这导致指令路由逻辑越过了应有的接缝（Seam），直接泄漏（Leakage）到了视图层。

解决方案： 提取出一个深模块（Deep Module）—— CommandDispatcher。它将暴露一个极简的接口供 UI 调用，而将所有繁杂的指令分类、路由分发（走本地 REST 还是走流式 API）的实现细节完全吸收。

  • 局部性 (Locality)： 所有的指令路由和拦截逻辑集中到了一个模块，不再散落各处。
  • 接口收缩： App.jsx 不再关心具体的命令内容，实现层吸收了所有的胶水代码。
  • 高杠杆率 (Leverage)： 统一的接口可以同时服务于普通的聊天输入框 (sendMessage) 和快捷按钮执行 (executeSlashCommand)。

最高优先级推荐

收拢指令调度器 (Collapse the Command Dispatcher)

刚刚为了切断记忆压缩和流式对话的耦合，我们将具体的 /summarize 指令直接硬编码到了 React 的核心 UI 逻辑中。通过深化模块并重建接缝（Seam），我们可以在未来随时添加更多的本地指令，而不会让 App.jsx 不断膨胀。
