# 媚吻锋随身课堂 (MeiWenfeng Classroom)

> 极致轻量化的本地 Web 角色扮演与教学引擎 - 基于 Agentic 工具流与深度解耦的管道架构

---

## 📖 项目简介

媚吻锋随身课堂是一个完全独立于 SillyTavern 的本地 Web 角色扮演与教学应用。项目基于 Python (FastAPI) 后端与 React 前端构建，旨在通过构建深度的 `ResponsePipeline` 拦截管道与 `AgentExecutor` 智能体执行器，在保障前端流式打字机平滑体验的同时，实现稳定、安全的多轮本地工具调用（Agentic Tool Flow）和高效的 RAG（检索增强生成）系统。

- ✅ **流式对话渲染** - 内嵌角色设定，自动渲染灰色斜体动作描写
- ✅ **动态记忆衰减** - 本地 ChromaDB 支持，自动提取核心并按遗忘曲线衰减
- ✅ **沉浸式双栏布局** - 专为 VS Code 生态深度重构：专注核心对话区与右侧导师状态面板
- ✅ **多模型无缝切换** - 完全支持 DeepSeek 以及本地 Ollama (如 gemma-9b)，架构已预留 Claude/Gemini 接入空间
- ✅ **内网跨屏访问** - 动态 API 寻址与 IP 白名单防黑设计，支持手机端无缝使用

## 🚀 快速开始

### 前置要求

- **Node.js** (推荐 18+版本)
- **Python** (推荐 3.10+ 版本)
- **API 密钥**：获取你希望使用的在线模型 API Key (如 DeepSeek) 或者准备好本地模型服务 (如 Ollama)。
- **网络环境 (关键)**：在初次部署和模型/向量库运行期间，必须开启稳定的 **VPN/全局代理** 以接入海外网络，保证能够顺利连接并从 Hugging Face 等站点下载必要的模型与依赖组件。

### 第一步：克隆并配置

```bash
# 1. 准备项目代码
git clone https://github.com/EDvisasa/MeiWenfeng-Classroom.git
cd MeiWenfeng-Classroom
```



### 第二步：一键启动

双击根目录下的 **`start.bat`**。

脚本将自动：
1. 启动 FastAPI 后端服务（绑定 `0.0.0.0:12701`）。
2. 启动 Vite 前端开发服务器（绑定 `0.0.0.0:12703`）。

> 💡 **局域网访问提醒**：项目默认开启了 IP 白名单防护。您不仅可以在本机的 `http://localhost:12703` 访问，也可以在同一 WiFi 下的手机上，通过输入 `http://您的电脑IP:12703` 进行访问，恶意不明 IP 会被后端自动拦截。

### 第三步：进阶配置 (可选)

> 💡 **提示**：大模型的 API Key 和 URL 等基础参数可直接在网页 UI 界面的「🔌 API 设置」中配置，即刻生效。

如果您需要挂载自定义的人设卡片或本地图片路径，请参考项目根目录下的 `.env.example` 文件，将其复制并命名为 `.env`，然后配置以下环境变量：
- `PERSONA_SOURCE_DIR`：指定 `/update_persona` 读取外部原版人设卡片的绝对目录路径（例如您的外部游戏角色卡目录）。
- `IMAGE_DIR`：用于挂载本地角色图片的目录。
- `PERSONA_SIMPLIFIED_OUTPUT_DIR`：精简人设的外部输出同步目录。

## ⚙️ 核心功能解析

### 🤖 Agentic 工具流与管道架构 (New!)
彻底告别将网络请求、工具执行和副作用拦截揉捏在一起的“上帝函数”。系统现已升级为智能体系统架构：
- **底座协议层 (`LLMClientProtocol`)**：纯粹的物理层流式通信。
- **应用执行层 (`AgentExecutor`)**：包裹底层协议，负责状态机驱动的多轮思考 (`<thought>`) 与本地工具 (`execute_bash`, `read_file`, `grep_search`) 调用循环。内建严密的 AST 级防注入护栏，有效拦截恶意 Shell 命令和路径穿越攻击。
- **响应拦截层 (`ResponsePipeline`)**：字符级状态机实时拦截 XML 副作用标签（如 `<glossary>`, `[SYSTEM_PASS]`），同时保障向前端输送绝对纯净的 SSE 文本流，避免打字机卡顿或标签泄漏。
- **大文件 OOM 防护**：通过 `@文件` 语法读取本地文件时，自动截断超大文件（限制 20000 字符），并智能引导大模型使用 `grep_search` 或 `read_file` 工具探索剩余内容，彻底杜绝上下文溢出崩溃。

### 🖥️ 沉浸式双栏布局与智能 UI
- **动态 Token 环形指示器**：实时计算底层隐藏上下文（系统设定/RAG/记忆）与当前对话的 Token 消耗总和，接近模型上限时自动变色预警。
- **强壮的思考状态管理**：前端解析流已重构为“深模块 (Deep Module)”，精准追踪大模型流式输出生命周期与多轮工具调用时序，彻底解决思考块（Thinking Block）状态卡死或指示灯泄漏问题。

### 🧠 记忆系统与时间线重构
内置本地 ChromaDB 向量知识库。基于自研的遗忘曲线算法，系统会按天对历史记忆（精确、关键、模糊、里程碑、索引）进行衰减处理和提炼，解决传统扮演 LLM 动辄塞入几万字上下文导致崩溃或幻觉的问题。

### 📚 课程与大纲进度控制
引入右侧的全局“导师状态栏”，完全图形化展示当前课题、用户理解分数与任务节点。所有交互行为带来的进度改变直接回写至底层的 SQLite `course_progress` 表。

### ⚡ 内嵌 LiteLLM 代理网关与智能多模型流控
- **内建代理网关**：内置 LiteLLM 提供全链路代理，统一接管大模型的协议转换（Gemini/Claude 自动翻译为标准 OpenAI SDK 调用），实现后端代码零耦合。
- 完全支持 **OpenAI 协议兼容** (如 DeepSeek/Qwen，以及本地 Ollama)
- 已预留 **Claude 原生协议** 接口 (由网关自动翻译适配)
- 已预留 **Gemini 原生协议** 接口 (由网关自动翻译适配)

通过右上角的 `🔌 API 设置`，随时切换模型，应对不同知识深度的教学需求。网关不仅会主动向上游探测真实 `max_tokens` 限界，现已支持 **自定义最大上下文 Tokens** 功能，通过 UI 锁定本地模型（如 Ollama、llama.cpp）的上下文上限，避免错误覆盖。

### 🛡️ 内置数据监控大屏
不必再使用第三方数据库软件瞎猜！前端集成了一个专用的 `DatabaseViewer` 组件。点击界面上的数据库审查图标，就能在只读弹出层内实时审查所有对话日志和用户隐藏档案。

## 📋 斜杠命令速查

在对话输入框内输入以 `/` 开头的指令，即可触发后台服务：

| 命令 | 功能描述 |
|------|------|
| `/lesson` | 开始新课时，加载当前课时大纲，切换为授课模式 |
| `/submit` | 检验理解程度，AI 切入"考核模式"出题并等待用户回答 |
| `/reward` | 课后奖励，基于隐藏好感度解锁特殊剧情段落 |
| `/summarize` | 主动压缩记忆，触发衰减算法整合并存入 SQLite |
| `/prepare` | 将当前选中的讲义同步至本地向量知识库 |
| `/plan` | 读取历史进度，对课程大纲的未来路径进行自动重规划 |
| `/set_mission` | 设定最终的宏大教学目标（用法：`/set_mission [目标]`） |
| `/update_persona` | 从源目录更新全量人设并自动提炼为精简人设 |
